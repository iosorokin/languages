Один из моих пет проектов. <br>
Здесь предполагался хелпер для изучения иностранных языков по фильмам книгам и музыке <br>
В проекте я больше оттачивал архитектуру и проводил постоянный рефакторинг, чем пытался его запустить <br>
Местами будет прослеживаться частая смена мысли, местами несвязный код и так далее. Следует понимать, что это полигон для испытаний,а не чистовик для продакшена <br>
После наработки разных практик, я перенёс архитектуру на симфони в другой пет проект в закрытом репозитории. <br>

Что я вынес из этого проекта:
- разделение слоёв на Модель, Презентатор, Базу данных, Приложение, Инфраструктура.
- программирование системы через объекты-значения

Какие практики отсюда я вынес в другой пет проект:
- скрытая валидация объектов значений (все объекты-значения присваиваются по интерфейсу. Конструкторы объектов значений запривачены. У каждого объекта-значения есть метод валидации. После присвоения всех объектов значений сущность следует провалидировать. Специальный метод валидации проходит по всем значениям собирая ошибки из них.
